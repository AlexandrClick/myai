# Project MCP Playbook

[← Вернуться в gemini.md](./gemini.md)

В этом workspace используются три MCP-сервера с графовой моделью:
- `code-review-graph` — для экономичного по токенам ревью и получения минимального контекста.
- `codegraph` — для поиска символов, анализа вызовов (callers/callees), структуры файлов и быстрой навигации по архитектуре.
- `gitnexus` — для поиска с учётом процессов, контекста выполнения, анализа влияния изменений, безопасного переименования и работы с несколькими репозиториями.

Используйте графовые инструменты до широких сканов файлов. Предпочитайте минимально достаточный контекст. Читайте целые файлы только если граф недостаточен, устарел или противоречив.

## Основные правила работы

0. Всегда отвечать на русском языке  
1. Начинайте с MCP-графовых инструментов, а не с Grep/Glob/Read.  
2. Минимизируйте использование токенов: выбирайте точечные запросы по символам/задачам вместо широких.  
3. Перед изменением существующего символа выполняйте анализ влияния.  
4. Перед ревью или дебагом сначала получайте минимальный контекст, затем углубляйтесь.  
5. Если сервер сообщает об устаревшем или отсутствующем индексе — сообщите пользователю и выполните rebuild.  
6. Не воспринимайте граф как требования к продукту — используйте его для структуры кода, зависимостей, потока выполнения и оценки рисков изменений.  
7. Если инструменты дают противоречивые данные — берите наиболее специфичный результат и проверяйте через чтение файлов.
